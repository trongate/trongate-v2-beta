Regarding Table Names

When interacting with a MySQL or MariaDB database, you'll probably be using Trongate's DB class. Most methods in Trongate's DB class include an optional $target_table parameter.

When you don't pass a table name, as an argument, Trongate automatically uses the first URL segment to be the table name.

For example, if your URL is https://yoursite.com/products/display, the table name defaults to products.

To make your code more robust, it's a good practice to always pass the table name as an argument.

In Trongate v2, use $this->db->get(). If you see $this->model->get() in reference materials, that's v1 syntax.

Example

Let's assume that your website BASE_URL is set to https://example.com/.

Let's also assume that you have a module named members, and a database table that's also named 'members'.

Finally, let's assume that you have a manage() method, on the Members.php controller file. This method could be used to render a page for managing member records. The manage() method could be invoked by going to:

[code]https://example.com/members/manage[/code]

In this scenario, we can use Trongate's get() method to retrieve rows from the members database table. The code might look like this:

[code=php] public function manage() { // Fetch members records. $rows = $this->db->get('id'); $data = [ 'members' => $rows, 'view_file' => 'manage_members' ]; $this->template('public'); } [/code]

In the above code sample, the important line here is:

[code=php]$rows = $this->db->get('id');[/code]

This code executes the following SQL query:

[code]SELECT * FROM members ORDER BY id;[/code]

Even though, we only passed in one argument (the $order_by value), the table name of members is inferred from the first URL segment (the first URL segment, in this example, is 'members').

Explicit Table Name Declaration

The ability to be able to invoke SQL queries with such few characters is very much in alignment with Trongate's propensity for simplicity. However, if there is ever a scenario where our URLs change (perhaps through some kind of custom routing setup, our code would break!

Therefore, to ensure that your code remains future proof, it's a good idea to pass in the $target_tbl argument when invoking methods from Trongate's DB class.

For example:

[code=php] public function manage() { // Fetch members records. $rows = $this->db->get('id', 'members'); $data = [ 'members' => $rows, 'view_file' => 'manage_members' ]; $this->template('public'); } [/code]
