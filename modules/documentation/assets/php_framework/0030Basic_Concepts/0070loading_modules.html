Loading Modules

Two ways. Zero drama.

Modules can be loaded from the URL or from code. No service containers. No artisan bullschitt. No PSR baloney. Just PHP.

1. From the URL (the easy way)

Go to: https://yoursite.com/members/profile/88

Trongate runs:

[code=php] <?php class Members extends Trongate { function profile($user_id = 88) { // go nuts } } [/code]

2. From Code (the fun way)

Inside any controller:

[code=php] // Load and use a 'tax' module (with one line of code!) $vat = $this->tax->calculate($amount); [/code]

Inside any view:

[/code]

Passing Data? Just Do It.

The code snippet below demonstrates loading a module and passing in arguments, from a controller file:

[code=php] $this->tax->apply($amount, $country); [/code]

Here's an example of calling a modules from within a view file (perfectly acceptable, by the way!) and in this example we pass in an argument of $page_id.

[code=vf] <?= Modules::run('banners/show', $page_id) ?> [/code]
