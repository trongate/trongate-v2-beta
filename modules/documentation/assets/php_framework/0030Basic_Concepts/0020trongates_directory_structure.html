Trongate’s Directory Structure

engine/

The beating heart. Routing, DB, helpers, magic. Written in proper modern PHP.
Don't touch the files in this folder ...or else!

config/

Plain PHP files. No YAML. No ENV hell.
config.php • database.php • custom_routing.php • etc.

modules/

Your app lives here. One folder per feature. No sub-folders named “controllers”, because 2005 called — it wants its nesting back.

[code] modules/ users/ Users.php ← controller (the boss) Users_model.php ← optional views/ ← pure PHP templates welcome.php login.php [/code]

Trongate v2 killed the controllers/ subfolder. You’re welcome.

templates/

Pure PHP templates. No Blade. No Twig. No “learning a new syntax” tax.
If you know PHP → you know templating. Done.

public/

The only folder the internet sees. Drop your CSS, JS, images, index.php, and that one favicon you keep forgetting.

Trongate has zero dependencies and seven folders. By the way, Laravel needs 28,492 files just to say “hello world”.

Less structure. More building. That’s Native PHP.
