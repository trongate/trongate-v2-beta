Trongate's Directory Structure

engine/

The beating heart. Routing, DB, helpers, magic. All written in modern PHP.

Don't touch the files in this folder ...or else!

config/

Plain PHP files. No YAML. No ENV complexity.

config.php • database.php • custom_routing.php • etc.

modules/

Your app lives here. One folder per feature. No sub-folders named "controllers", because that's unnecessary nesting.

[code] modules/ users/ Users.php ← controller (the boss) Users_model.php ← optional views/ ← pure PHP (no template engine) welcome.php login.php [/code]

Trongate v2 killed the controllers/ subfolder. You're welcome.

templates/

Pure PHP templates. No template languages. No "learning a new syntax" tax.

If you know PHP → you know templating. Done.

public/

The only folder the internet sees. Drop your CSS, JS, images, index.php, and that one favicon you keep forgetting.

Trongate has no third-party dependencies.
