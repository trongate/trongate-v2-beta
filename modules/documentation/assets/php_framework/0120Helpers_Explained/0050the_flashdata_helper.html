Flashdata Helpers

Trongate includes flashdata helper functions for displaying one-time messages. Flashdata is session data that is available only for the next request before being automatically removed. This is useful for informational messages like "The record was successfully created". The following functions are available from the Flashdata Helper:

  • set_flashdata() Store a one-time message in the session
  • flashdata() Display and clear flashdata message

Setting Flashdata

Flashdata can be set using the set_flashdata() function. You can pass your message directly into the function:

[code=php] set_flashdata("The record was successfully updated"); [/code]

Typical Usage Pattern: Flashdata is commonly set after a form submission or data operation, followed by a redirect to another page where the message will be displayed:

[code=php] // After successfully creating a record set_flashdata("The record was successfully created"); redirect("admin/records"); [/code]

Displaying Flashdata

Flashdata can be displayed by calling flashdata() from within a view file:

[code=php] flashdata(); [/code]

Automatic Removal: The message will be displayed once and then automatically removed from the session. If you call flashdata() again on a subsequent page load, nothing will be displayed.

Customizing Flashdata Display

By default, flashdata messages will be displayed as green text within a paragraph. However, you can change the format of your flashdata by adding optional opening and closing tags as arguments. For example:

[code=php] flashdata('<p style="color: lime; font-weight: bold;">', '</p>'); [/code]

Or use custom CSS classes:

[code=php] flashdata('<div class="alert alert-success">', '</div>'); [/code]

Global Configuration: You can define FLASHDATA_OPEN and FLASHDATA_CLOSE constants in your configuration to set default HTML wrappers for all flashdata messages throughout your application:

[code=php] define('FLASHDATA_OPEN', '<div class="alert alert-success">'); define('FLASHDATA_CLOSE', '</div>'); [/code]

When these constants are defined, all flashdata messages will automatically use these wrappers unless you explicitly pass different opening and closing tags to the flashdata() function.
