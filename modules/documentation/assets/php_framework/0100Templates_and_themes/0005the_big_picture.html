The Big Picture

You already know that view files are just PHP files. HTML + <?= $var ?> = done. No template engine. No drama.

Now meet the next level: templates and themes.

Templates

Templates are full-page PHP files that define the complete HTML skeleton – <!DOCTYPE> to </html>. Header, footer, navigation, scripts – the stuff that’s identical on every page.

One line loads everything and injects your view:

[code=php] $this->template('public', $data); [/code]

The $data array is automatically available in both the template and the view. Zero extra code.

Templates vs Views (definitions)

  • Templates – the wrapper (full HTML page)
  • Views – the changing content inside it

Templates load views. Views never load templates. Clean separation. Done.

Where Templates Live

Everything lives in the templates module at the root of your app:

The terms template module and template directory refer to the same thing: the templates directory in a Trongate application.

Themes – Templates on Steroids

A theme is a complete visual package: template file + CSS + JS + images + fonts, all living together in public/themes/.

Register once in config/themes.php:

[code=php] [ 'dir' => 'default_admin/blue', 'template' => 'admin.php', ], 'bootstrappy' => [ 'dir' => 'bootstrappy/dark', 'template' => 'bootstrappy.php', ], ]; define('THEMES', $themes); [/code]

Then load exactly like a normal template:

[code=php] $this->template('bootstrappy', $data); [/code]

Trongate grabs the template from the theme folder, sets THEME_DIR for your assets, and you’re live. Colour variations? Just change the sub-folder. Instant new skin!

Why This Wins

  • 100% Native PHP – no new syntax
  • DRY layouts with zero effort
  • Portable themes – copy folder, register, done
  • No forced CSS/JS frameworks
  • Switch look & feel at runtime (admin vs member vs guest)

Templates provide consistent structure. Themes provide consistent structure plus consistent visuals packaged together.
