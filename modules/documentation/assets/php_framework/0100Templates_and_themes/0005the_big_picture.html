The Big Picture

You already know that view files are just PHP files. HTML + <?= $var ?> = done. No template engine. No drama.

Now meet the next level: templates.

Templates

Templates are full-page PHP files that define the complete HTML skeleton – <!DOCTYPE> to </html>. Header, footer, navigation, scripts – the stuff that’s identical on every page.

One line loads everything and injects your view:

[code=php] $this->templates->public($data); [/code]

The $data array is automatically available in both the template and the view. Zero extra code.

Templates vs Views (definitions)

  • Templates – the wrapper (full HTML page)
  • Views – the changing content inside it

Templates load views. Views never load templates. Clean separation. Done.

Where Templates Live

In Trongate, everything is made of modules. That includes templates.

Therefore,all of your templates exist within the modules directory.

Default Templates

Sure enough, Trongate comes with a module called 'templates' and this your default location for templates.

The templates module is contructed like any ordinary Trongate module. That includes:

  1. One controller file (Templates.php).
  2. A views directory for storing presentation files.

In addition, the templates module can contain other assets that HTML templates may require, such as; stylesheets, icons, fonts etc.
