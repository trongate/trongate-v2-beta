Template Partials

Partials are reusable chunks of template code: headers, footers, navbars, sidebars, whatever.

Trongate supports them. You can use them. They work exactly like you'd expect.

Example: Template Without Partials

Here's a basic template with footer markup directly inside:

[code=html] <!DOCTYPE html> <html lang="en"> <head> <base href="<?= BASE_URL ?>"> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title><?= $page_title ?? 'My Site' ?></title> </head> <body> <?= Template::display($data) ?> <footer> <p>&copy; <?= date('Y') ?> <?= OUR_NAME ?></p> <p>Contact: info@example.com</p> </footer> </body> </html> [/code]

Example: Template With Partials

Same template, but the footer is loaded from a partial:

[code=html] <!DOCTYPE html> <html lang="en"> <head> <base href="<?= BASE_URL ?>"> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title><?= $page_title ?? 'My Site' ?></title> </head> <body> <?= Template::display($data) ?> <?= Template::partial('partials/footer', $data) ?> </body> </html> [/code]

The footer partial (templates/views/partials/footer.php):

[code=html] <footer> <p>&copy; <?= date('Y') ?> <?= OUR_NAME ?></p> <p>Contact: info@example.com</p> </footer> [/code]

Same result, reusable markup.

How Partials Work

Partials live in templates/views/ (or subfolders).

Load one from your template file with Template::partial():

[code=php] <?= Template::partial('footer') ?> [/code]

NOTE: The partial() method has the ability to pass an array of $data to a partial. For example:

[code=php] <?= Template::partial('footer', $data) ?> [/code]

The $data array gets automatically extracted, making variables available directly in the partial.

Partials are a common pattern in other frameworks, which is why Trongate supports them. However, the preferred Trongate approach for reusable UI is to load entire modules directly from your templates or views.

See: Loading modules from templates/views
