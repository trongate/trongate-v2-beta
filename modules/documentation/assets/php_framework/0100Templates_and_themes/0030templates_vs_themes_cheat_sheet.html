Templates vs Themes – Cheat Sheet

Everything you need to know, at a glance.

The Quick Comparison

Feature Templates Themes
Where they live templates/views/ public/themes/your_folder/
Contains assets? No Yes—everything
Portable? Copy-paste code Copy entire folder → done
Color variations Manual CSS changes Just swap sub-folder
THEME_DIR available? No Yes
Best for Simple sites, quick prototypes Admin areas, client projects, reusable designs

Loading Pattern (Same Method, Different Results)

Regular Template:

[code=php] $this->template('public', $data); // Loads: templates/views/public.php [/code]

Theme:

[code=php] $this->template('admin', $data); // Checks: config/themes.php // Loads: public/themes/default_admin/blue/admin.php // Sets: THEME_DIR constant [/code]

Quick Decision

Need multiple color schemes? → Theme

Want to copy design to another project? → Theme

Just need consistent layout? → Template

99% of the time you'll use regular templates for public-facing pages and themes for anything that needs a full design system (admin, member area, SaaS dashboard).

The Bottom Line

  • Both use the same template() method.
  • Both inject views the same way.
  • Templates = structure only.
  • Themes = structure + complete visual package.
  • Switch between them anytime—no drama.
