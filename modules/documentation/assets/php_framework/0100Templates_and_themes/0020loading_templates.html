Loading Templates

One method. Zero surprises.

From any controller, load a template like this:

[code=php] $this->template('public', $data); [/code]

That's it. Trongate finds the right method in Templates.php, loads the matching view file from templates/views/, injects your view content, and renders the page.

What You Usually Pass

[code=php] $data['view_file'] = 'welcome'; // required – the view to inject $data['view_module'] = 'welcome'; // optional – defaults to current module $data['page_title'] = 'Hello World'; // whatever you want in the template $this->template('public', $data); [/code]

Always set view_module explicitly. Future-you will thank present-you when routing gets spicy.

Behind the Scenes

When you call $this->template('public', $data), Trongate:

  • Looks for Templates::public($data) in the Templates class
  • Runs the extra include magic (top + bottom assets)
  • Calls helpers-Utilities_Helpers-load() → pulls templates/views/public.php
  • Inside that file, Template::display($data) injects your actual view content

What a Template File Looks Like

Here's a typical template (public.php):

[code=html] <!DOCTYPE html> <html lang="en"> <head> <base href="<?= BASE_URL ?>"> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <link rel="stylesheet" href="css/trongate.css"> <title><?= $page_title ?? WEBSITE_NAME ?></title> </head> <body> <header> <nav> <?= anchor(BASE_URL, 'Home') ?> <?= anchor('about', 'About') ?> </nav> </header> <main class="container"> <?= Template::display($data) ?> </main> <footer> <div>&copy; <?= date('Y') . ' ' . OUR_NAME ?></div> </footer> </body> </html> [/code]

Template::display($data) is where your view content appears. Everything else stays consistent across pages.

The class_reference-The_Template_Class-display() method, as shown above, is responsible for loading and displaying view files from module directories.

Quick Example

[code=php] class Dashboard extends Trongate { function index() { $data['view_file'] = 'dashboard_home'; $data['view_module'] = 'dashboard'; $data['page_title'] = 'My Dashboard'; $this->template('admin', $data); // loads the admin theme instantly } } [/code]

One line. Full page. Theme or regular template - works exactly the same.
