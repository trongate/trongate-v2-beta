Building Your Own Theme

You've seen how themes load. Now make one. It's stupidly simple.

Basic Theme (No Shared Assets)

Let's start with the simplest possible theme structure.

Step 1 – Make the Folder

Create a folder inside public/themes/:

[code] public/themes/ acme_dashboard/ dark/ light/ [/code]

Step 2 – Add Your Template File

Create the template file inside the dark/ folder:

public/themes/acme_dashboard/dark/acme.php

[code=html] <!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <base href="<?= BASE_URL ?>"> <link rel="stylesheet" href="<?= THEME_DIR ?>css/style.css"> <?= $additional_includes_top ?? '' ?> <title><?= $page_title ?? 'Acme Dashboard' ?></title> </head> <body> <?= Template::display($data) ?> <script src="<?= THEME_DIR ?>js/app.js"></script> <?= $additional_includes_btm ?? '' ?> </body> </html> [/code]

All assets live inside dark/. THEME_DIR points directly to them.

Step 3 – Register It

Open config/themes.php and add:

[code=php] <?php $themes = [ 'acme_dark' => [ 'dir' => 'acme_dashboard/dark', 'template' => 'acme.php', ], 'acme_light' => [ 'dir' => 'acme_dashboard/light', 'template' => 'acme.php', ], ]; define('THEMES', $themes); [/code]

Step 4 – Handle It in Templates.php

Add a method in templates/controllers/Templates.php:

[code=php] public function acme_dark(array $data): void { $data['additional_includes_top'] = $this->build_additional_includes($data['additional_includes_top'] ?? []); $data['additional_includes_btm'] = $this->build_additional_includes($data['additional_includes_btm'] ?? []); load('acme_dark', $data); } [/code]

Repeat for acme_light() if needed.

Step 5 – Use It

[code=php] $this->template('acme_dark', $data); [/code]

Done. Refresh. New theme.

Advanced: Using Shared Assets

What if multiple color variations share fonts, icons, or base styles? Answer: Use a shared/ folder.

Updated Structure

[code] public/themes/ acme_dashboard/ shared/ css/ fonts.css reset.css js/ utils.js dark/ css/ style.css ← dark-specific styles only acme.php light/ css/ style.css ← light-specific styles only acme.php [/code]

Set Up Shared Path in Templates.php

Establish the $shared_dir variable in your template method:

[code=php] public function acme_dark(array $data): void { // Shared assets live one level up from the current theme variation $data['shared_dir'] = dirname(rtrim(THEME_DIR, '/')) . '/shared/'; $data['additional_includes_top'] = $this->build_additional_includes($data['additional_includes_top'] ?? []); $data['additional_includes_btm'] = $this->build_additional_includes($data['additional_includes_btm'] ?? []); load('acme_dark', $data); } [/code]

Now $shared_dir is available in your template.

Use It in Your Template

public/themes/acme_dashboard/dark/acme.php

[code=html] <!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <base href="<?= BASE_URL ?>"> <!-- Shared assets --> <link rel="stylesheet" href="<?= $shared_dir ?>css/fonts.css"> <link rel="stylesheet" href="<?= $shared_dir ?>css/reset.css"> <!-- Variation-specific assets --> <link rel="stylesheet" href="<?= THEME_DIR ?>css/style.css"> <?= $additional_includes_top ?? '' ?> <title><?= $page_title ?? 'Acme Dashboard' ?></title> </head> <body> <?= Template::display($data) ?> <script src="<?= $shared_dir ?>js/utils.js"></script> <script src="<?= THEME_DIR ?>js/app.js"></script> <?= $additional_includes_btm ?? '' ?> </body> </html> [/code]

Shared assets load once. Variation-specific styles override as needed. Clean and DRY.

  • Start simple—use a basic structure until you actually need shared assets.
  • Put fonts, icon sets, and CSS resets in shared/.
  • Only override what changes between variations.
  • Want a single-variation theme? Skip sub-folders: 'dir' => 'my_theme'.
  • Share themes between projects? Just copy the folder and register it.

Go build something beautiful.
